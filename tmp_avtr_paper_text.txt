A VTR-HRAG: Agentic Verifiable Tree-Reasoning Hybrid
Retrieval-Augmented Generation
A proof-carrying, IRT-calibrated architecture for exhaustive and auditable document intelligence
Shivansh Mukhia
30 June 2026
Abstract
Retrieval-augmented generation (RAG) systems usually retrieve a small set of top-ranked chunks and ask
a language model to answer from those chunks. That static pipeline is useful for simple lookup, but it is not
enough for high-recall document intelligence tasks requiring complete evidence discovery, exact citations,
table recovery, multi-document comparison, uncertainty reporting, and resistance to unsupported synthesis.
This paper proposes A VTR-HRAG, an Agentic Verifiable Tree-Reasoning Hybrid Retrieval-Augmented
Generation architecture that extends the earlier Verifiable Retrieval System and VTR-HRAG designs with
a controlled agent layer, proof-carrying retrieval contracts, and Item Response Theory (IRT)-calibrated
evaluation.
The proposed system preserves the original VTR-HRAG rule that internal tree summaries can route
retrieval but cannot prove final claims. A VTR-HRAG adds specialized agents for planning, lexical retrieval,
dense retrieval, table retrieval, tree descent, contradiction search, budget control, and verification. The
agents do not freely invent answers; they can only propose retrieval actions, evidence expansions, or verifier
decisions. A final answer is valid only when each atomic claim is supported by leaf-level evidence such as
paragraph spans, OCR regions, table cells, captions, or page regions.
The paper formalizes hybrid rank fusion, coverage-aware selection, agentic expected-value-of-information
(EVI) control, theoretical recall improvements from non-redundant retrieval branches, verification error
bounds, latency-cost tradeoffs, estimation-error analysis, and IRT-based evaluation. It also proposes an IRT-
fitted benchmark design with binary and partial-credit outcomes for retrieval completeness, citation precision,
table accuracy, abstention behavior, and unsupported-claim rate. The central thesis is conditional but strong:
agentic RAG can outperform static hybrid RAG when agents are constrained by provenance, leaf-only proof
rules, adaptive budgets, and calibrated evaluation rather than allowed to become an unconstrained reasoning
loop.
Keywords: retrieval-augmented generation, Agentic RAG, hybrid search, BM25, dense retrieval, tree chunks,
GraphRAG, RAPTOR, RRF, IRT, evaluation, citation grounding, provenance, hallucination detection, table
retrieval, evidence verification

1. Source Verification and Scope
This paper is a consolidated and upgraded research draft. It extends two supplied prior designs: the first Verifiable
Retrieval System draft, which focused on exhaustive low-latency PDF retrieval with provenance, exact evidence,
hybrid retrieval, span-level citation, and verification; and the second VTR-HRAG draft, which upgraded that
design with typed evidence trees, contextualized tree chunks, reasoning-guided retrieval, and the rule that
summaries guide while leaves prove.
External references were checked from primary or near-primary sources where possible. The literature base
includes the original RAG paper, Dense Passage Retrieval (DPR), BM25/RRF foundations, ColBERT, RAPTOR,
GraphRAG, Contextual Retrieval, Self-RAG, Corrective RAG, reasoning Agentic RAG surveys, RAGAS, ARES,
RAGTruth, BEIR, HotpotQA, MuSiQue, KILT, and IRT-based LLM evaluation. Empirical values plotted in
Figure 3 are source-reported values from Anthropic’s Contextual Retrieval article. Other performance plots are
explicitly marked as theoretical or illustrative simulations and are not presented as measured benchmark results.
2. Introduction
Document intelligence is not simply semantic search. Users often ask questions such as “find every clause related
to cancellation,” “compare all policy contradictions,” “recover the exact numeric values in every relevant table,”
or “summarize the corpus while citing every claim.” A static RAG pipeline that embeds chunks, retrieves top-k,
and generates an answer can fail silently: the missing passage is invisible, the citation may be imprecise, and a
fluent answer can conceal omission.
The earlier VTR-HRAG design already addressed this problem by combining lexical retrieval, dense retrieval,
contextual chunks, table retrieval, tree summaries, graph traversal, coverage selection, reranking, and claim
verification. Its key discipline was: internal summaries may guide retrieval, but only leaf evidence can prove
claims. A VTR-HRAG keeps this rule and adds an agentic control layer. The purpose of the agent layer is not
to make generation more theatrical. It is to make retrieval more adaptive, evidence search more complete, and
evaluation more calibrated.
The research question is:
Can Agentic RAG be fused with verifiable tree-hybrid RAG in a way that improves expected retrieval
coverage and answer support without exploding hallucination, latency, or evaluation ambiguity?
This paper argues yes, under four constraints:
1. Agents must operate over typed tools and typed evidence objects, not free-form hidden reasoning.
2. Generated summaries, agent notes, and planner outputs may route search but may not be cited as proof.
3. Agentic expansion must be controlled by expected value of information and IRT-estimated difficulty.
4. Evaluation must separate retrieval, evidence selection, generation, verification, abstention, latency, and cost.
3. Background and Research Gap
3.1. Classic and Hybrid RAG
Lewis et al. introduced RAG as a way to combine parametric sequence-to-sequence models with non-parametric
dense retrieval memory for knowledge-intensive NLP tasks [1]. DPR showed that learned dense dual-encoder
retrieval can outperform a strong BM25 baseline on open-domain QA in several settings, with reported top-20
passage retrieval improvements of 9–19 absolute points over Lucene-BM25 [2]. However, private document
intelligence differs from open-domain QA. Exact identifiers, rare legal phrases, table labels, scanned text, page
layout, and repeated boilerplate can make dense-only retrieval unreliable.
BM25 remains a strong lexical baseline for exact terms and rare tokens [3]. Reciprocal Rank Fusion (RRF)
is a practical method for merging rankings from multiple retrieval systems without requiring score calibration
[4]. ColBERT demonstrates that late interaction can preserve token-level matching strength while reducing

query-time cost compared with full cross-encoder reranking [5]. These results motivate a hybrid strategy: lexical,
dense, late-interaction, reranking, and structural expansion should be treated as complementary retrieval surfaces,
not mutually exclusive choices.
3.2. Tree, Graph, and Contextual Retrieval
RAPTOR constructs recursive clusters and summaries over text chunks to support retrieval at multiple abstraction
levels. The authors report strong improvements on reasoning-heavy QA, including a 20-point absolute accuracy
improvement over the previous best QuALITY benchmark result when paired with GPT-4 [ 6]. GraphRAG
addresses global corpus questions by building entity graphs and community summaries, showing improvements
over naive RAG on corpus-wide sensemaking tasks [7]. Anthropic’s Contextual Retrieval adds short chunk-
specific context before embedding and BM25 indexing; its article reports reductions in top-20 retrieval failure
from 5.7% to 3.7% with contextual embeddings, to 2.9% when contextual BM25 is added, and to 1.9% when
reranking is added [8].
These methods reveal a pattern: long-document RAG improves when retrieval can use higher-level structure.
The danger is that summaries can compress away exceptions, qualifiers, and numbers. Therefore, A VTR-HRAG
treats tree and graph summaries as routing objects only. They can point to evidence, but final claims must
descend to leaves.
3.3. Self-Corrective and Agentic RAG
Self-RAG trains models to retrieve and critique using reflection tokens, improving factuality and citation behavior
relative to static retrieval in reported experiments [ 9]. Corrective RAG (CRAG) adds a lightweight retrieval
evaluator that decides whether retrieved documents are useful and triggers corrective actions when retrieval is
weak [10]. Recent surveys of reasoning Agentic RAG categorize the field into predefined reasoning workflows
and agentic reasoning workflows, highlighting planning, tool use, self-reflection, and dynamic retrieval as key
directions for real-world tasks [11, 12].
The open gap is not another agent loop. The gap is a controlled fusion of agents with verifiable evidence
discipline. Without proof constraints, agents can amplify retrieval errors, over-search irrelevant branches, or
confidently summarize unsupported material. A VTR-HRAG is designed to avoid that failure.
3.4. Evaluation Gap
RAG evaluation is still immature. RAGAS proposes reference-free metrics such as faithfulness, context precision,
and answer relevancy [13]. ARES automates RAG evaluation using synthetic training data, lightweight judges,
and prediction-powered inference with limited human labels [14]. RAGTruth provides nearly 18,000 manually
annotated RAG responses with word-level hallucination labels across domains [15]. BEIR offers a heterogeneous
retrieval benchmark across 18 datasets and shows BM25 remains robust while reranking and late-interaction
methods often perform best at higher computational cost [ 16]. HotpotQA and MuSiQue stress multi-hop
reasoning, with HotpotQA providing sentence-level supporting facts and MuSiQue-Ans containing 25K 2–4 hop
questions designed to reduce shortcut solving [18, 19].
However, raw averages can hide benchmark difficulty and model specialization. IRT provides a way to
separate system ability from item difficulty and discrimination. Recent LLM evaluation work argues that IRT
can make evaluation more data-efficient and capability-aware; LEGO-IRT reports stable capability estimates
using 3% of evaluation items and up to 10% lower estimation error by leveraging structural knowledge [ 20].
A VTR-HRAG therefore uses IRT not as decoration but as a calibration layer for benchmark scoring and adaptive
retrieval budgets.

7.5. Verification Error Bound
Let C− be the set of unsupported candidate claims produced before verification. Let the verifier’s false-acceptance
rate on unsupported claims be αv = P(ˆy = supported | c ∈ C−). If all final claims pass through the verifier,

9. IRT-Calibrated Evaluation Model
9.1. Why IRT Instead of Raw Accuracy
Raw accuracy treats every query as equally difficult. That is wrong for RAG. A single invoice-number lookup,
a two-hop comparison, a scanned-table extraction, and a corpus-wide contradiction query are not equal. IRT
models separate system ability from item difficulty and item discrimination.

For ranking metrics such as nDCG and recall@k, use paired bootstrap over queries. For p95 latency, use

10.2. Extraction and OCR Error
Let Pparse be the probability that a relevant leaf is correctly extracted and indexed. Even a perfect retriever
cannot recover evidence that was never extracted:
P(g recovered) = Pparse(g)Pindex(g)Pretrieve (g)Pselect(g). (28)
Thus, evaluation must include parser-level gold checks for table cells, OCR spans, reading order, and bounding
boxes. A strong answer model cannot compensate for missing source objects.
10.3. Gold Label Incompleteness
Exhaustive retrieval benchmarks have a special problem: the gold set may itself be incomplete. A VTR-HRAG
uses pooled judging and capture-recapture style auditing. If two independent annotation passes find n1 and n2
relevant leaves with overlap m, a rough estimate of total relevant leaves is:
ˆN = (n1 + 1)(n2 + 1)

This estimate is only a diagnostic, but it exposes when “recall” is being measured against an incomplete gold set.

10.4. Verifier Error
The verifier itself must be evaluated. Report false accept rate, false reject rate, numeric-check error, table

11.3. Ablation Plan
A VTR-HRAG must prove each added component. The required ablations are:
1. Dense-only top- k RAG.
2. BM25-only retrieval.
3. BM25 + dense + RRF.
4. Hybrid + reranker.
5. Hybrid + contextual chunks.
6. Hybrid + table-aware retrieval.
7. Hybrid + tree-summary routing.
8. VTR-HRAG without agents.
9. A VTR-HRAG with planner and critic agents only.
10. A VTR-HRAG with full agent layer but no verifier.
11. Full A VTR-HRAG with verifier and IRT budget controller.
The “no verifier” ablation is essential because agentic retrieval can increase answer fluency while also increasing
unsupported synthesis if not checked.
12. Algorithms
12.1. Index-Time Procedure
1. Classify each document as born-digital, scanned, or mixed.
2. Extract text, OCR spans, pages, blocks, tables, rows, cells, figures, captions, metadata, and layout coordinates.
3. Assign provenance metadata and stable hashes to every object.

--- PAGE 16 ---
A VTR-HRAG Research Paper — Shivansh Mukhia 16
4. Create L0 leaves and L1 passage chunks with leaf pointers.
5. Build section, page, document, topic, and corpus-map internal nodes.
6. Generate constrained contextual text for each chunk using only local document structure.
7. Index raw leaves, contextual chunks, dense vectors, table objects, tree summaries, and optional graph entities.
8. Validate that random citations reconstruct exact page regions and text spans.
12.2. Query-Time Procedure
1. Planner classifies query intent and decomposes into subclaims.
2. Budget controller estimates difficulty and initial budget.
3. Lexical, dense, contextual, table, tree, and graph agents retrieve candidates.
4. Fusion layer combines rankings with adaptive RRF and coverage-aware selection.
5. Tree-router descends from summaries to leaf descendants.
6. Critic checks missing subclaims, weak evidence, contradiction risk, and table/numeric gaps.
7. If EVI is positive and budget remains, generate new subqueries and repeat retrieval.
8. Proof collector fetches leaf evidence and provenance.
9. Generator drafts answer from proof set only.
10. Verifier extracts atomic claims and labels support as supported, partial, unsupported, contradicted, or
unverifiable.
11. Unsupported claims are removed, downgraded to uncertainty, or moved to abstention notes.
12. Return cited answer, evidence coverage notes, and audit log.
13. Security, Prompt Injection, and Governance
Retrieved documents are untrusted input. The generator must never treat document content as system instructions.
A VTR-HRAG uses four defenses:
1. Instruction separation: retrieved text is evidence, not policy.
2. Typed tools: agents can call retrieval and verification tools but cannot modify hidden system rules.
3. ACL propagation: access-control tags stay attached from ingestion to answer.
4. Injection flagging: prompt-like strings inside documents receive risk penalties and can be quoted only as
evidence when relevant.

16. Conclusion
A VTR-HRAG is a controlled fusion of Agentic RAG and verifiable tree-hybrid RAG. The proposed architecture
does not claim that agents are automatically better. It claims that agents become useful when they are constrained
by provenance, typed retrieval tools, coverage objectives, leaf-only proof rules, verification, and IRT-calibrated
evaluation.

--- PAGE 18 ---
A VTR-HRAG Research Paper — Shivansh Mukhia 18
Mathematically, A VTR-HRAG can dominate a static hybrid baseline when its action set includes the baseline
and the controller chooses actions by expected utility. The union-recall model shows why non-redundant retrieval
branches reduce evidence miss probability. The IRT model provides a principled way to measure system ability
while accounting for query difficulty, table extraction, multi-hop reasoning, abstention behavior, and verifier
reliability. The result is not just a stronger RAG pipeline. It is an evidence engine: it plans, searches, verifies,
cites, abstains, and measures uncertainty.
